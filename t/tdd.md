# Test-driven Development

`Test-driven Development` (TDD) is a software development technique where you write automated tests before writing the application code.

1. Write a test that describes the desired behaviour.
2. Run the test and watch it fail.
3. Write the minimum code needed to make the test pass.
4. Refactor the code while keeping the test passing.
