# Test-driven Development

`Test-driven Development` (TDD) is an [Agile](../a/Agile.md) software development methodology where you write automated tests before writing the application code.

1. Write a test that describes the desired behaviour.
2. Run the test and watch it fail – **red**.
3. Write the minimum code needed to make the test pass – **green**.
4. Improve the code while keeping the test passing – **refactor**.

Originated during the late 1990s as part of Extreme Programming.

evolved into BDD
